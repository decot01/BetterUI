# BetterUI

Библиотека React-компонентов с продвинутым UX.

**Почему наш UX подход сильный:** мгновенная обратная связь при наведении (обводка и заливка реагируют без задержки), увеличенная зона наведения (`.raycast`) — проще попасть по кнопке, меньше промахов. Единые варианты через `SubClass` дают консистентный интерфейс: пользователь сразу понимает, что кликабельно и как элемент себя ведёт.

**Мы не используем ИИ** — весь код и документация написаны вручную.

## Кнопка (Button)

Ссылка или кнопка: иконка (SVG), картинка, текст. Текст поддерживает HTML. Внешний вид — через `SubClass`. Подключите стили для `.button`, `.stroke`, `.fill`, `.raycast` и вариантов по `SubClass`.

### Как устроена кнопка

Разметка внутри компонента (порядок и классы):

```
.button[.SubClass]
├── .stroke     — обводка, при ховере на .raycast показывается и расширяется
├── .fill       — заливка фона (градиент или цвет по SubClass)
├── .raycast    — невидимая зона наведения, стили через :has(.raycast:hover)
├── .containerIcon        — опционально, если передан Svg (HTML иконки)
├── .containerIcon.imageIcon — опционально, если передан ImageUrl
└── .container.textHelper — опционально, если передан Text (через TextHelper)
```

Иконка → `.containerIcon`, картинка → `.containerIcon.imageIcon`, текст → **TextHelper** (`.container.textHelper`).

### Пропсы

| Проп       | Тип          | Обязательный | Описание                                                |
| ---------- | ------------ | ------------ | ------------------------------------------------------- |
| `Url`      | `string`     | да           | Адрес ссылки (при `asButton` можно `"#"`).              |
| `SubClass` | `string`     | нет          | Класс варианта: `donate`, `logo`, `about`, `login`, `social` и т.д. |
| `Text`     | `string`     | нет          | Подпись. Поддерживает HTML (`<span>`, `<br/>`).         |
| `Svg`      | `string`     | нет          | HTML иконки, например `<svg>...</svg>`.                 |
| `ImageUrl` | `string`     | нет          | URL картинки (для `logo` и подобных).                   |
| `asButton` | `boolean`    | нет          | Рендерить `<button>` вместо `<a>`.                      |
| `onClick`  | `() => void` | нет          | Обработчик клика при `asButton={true}`.                |

### Пример

Один пример, который показывает и ссылку, и вариант `donate`, и иконку (SVG), и текст с HTML — полная структура использования:

```tsx
import { Button } from './components/Button'

// Иконка — строка с HTML (например, SVG сердца)
const HeartSvg = `<svg viewBox="0 0 22 17" xmlns="http://www.w3.org/2000/svg" width="22.5" height="17"><path fill-rule="evenodd" d="M10.1863 17C5.47623 15.6278 -0.978755 11.92 0.124708 4.82672C1.11674 -1.54371 8.81923 -0.395855 11.0008 1.45942C13.1824 -0.395855 20.8848 -1.54371 21.876 4.82672C22.9794 11.92 16.5272 15.6278 10.1863 17Z" /></svg>`

<Button
  Url="/donate"
  SubClass="donate"
  Svg={HeartSvg}
  Text='<span style="letter-spacing: 1px;">T</span>he bu<span style="letter-spacing: 0.3px;">t</span>ton'
/>
```

Что здесь происходит: рендерится `<a class="button donate">` с внутренними слоями `.stroke`, `.fill`, `.raycast`, затем блок с иконкой (из `Svg`), затем блок с текстом (из `Text` через TextHelper, с поддержкой HTML). Вариант `donate` даёт синий градиент и анимацию обводки при наведении.
